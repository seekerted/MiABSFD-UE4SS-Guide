Nexus Mods and Vortex

In an effort to make it easier to look for and install mods, I've started uploading mods to Nexus Mods and also submitted a game extension to create support using Vortex, their mod manager.

Installing Vortex and Adding MiABSFD Support

At the time this was written, Vortex hasn't added my game extensions as one if its defaults yet, so you have to manually add it to Vortex.

  1. Download Vortex
  2. Download the support extension manually.
  3. In Vortex > Extensions, drag the downloaded zip file of the extension onto the Drop File(s).
  4. Restart Vortex
  5. In Vortex > Games, search for Made in Abyss and click Manage on it. Agree to admin prompts if needed.

You're done!

Installing Mods

Once you start managing Made in Abyss you can head to Vortex > Mods and drag / drop any mod .zip files to handle them.

  1. Once you've finalized your list of mods, just before you run the game, hit Deploy Mods. The way Vortex works is it keeps the game folder untouched, so that the files can be removed anytime. Deploying mods is where it creates symlinks to the game folder to actually add the files onto the game folder.
  2. If on the upper-right hand of the screen, it says "Deployment requires elevation," hit Elevate. This is for the creation of those symlinks.
  3. You can start playing the game by clicking on the play button on the upper-left.

Creating Mods that are compatible with Vortex

The game extension supports three kinds of mods: basic pak mods, UE4SS Lua mods, and UE4ss Blueprint mods. All mods must be inside a .zip file.

If these are followed, your mods should be handled by the game extension just fine.

.pak mods

Simply put your .pak files inside the .zip file, in the root.

UE4SS Lua Mods

Ensure that the .zip file contains at least the following:

Mods/modname/Scripts/main.lua
Mods/modname/enabled.txt

UE4SS Blueprint Mods

Similar to the .pak mods, except that it must be placed inside a LogicMods folder.
